# 项目范围

## 项目定位

本项目为视觉 SLAM 移动机器人树莓派上位机工程。目标是在原 x86 架构上位机仓库 `YQuark/Ros2_Slam` 的基础上，完成面向树莓派 4B 的迁移、裁剪、重构和工程化适配。

仓库地址：[YQuark/Ros2_Slam](https://github.com/YQuark/Ros2_Slam)，当前工作分支为 `ARM`。

## 平台约束

| 项目 | 规格 |
|------|------|
| 上位机 | Raspberry Pi 4B 4GB |
| 操作系统 | Ubuntu 22.04 64-bit (arm64) |
| ROS2 版本 | Humble |
| CPU | 4×Cortex-A72 @ 1.5GHz |
| 内存 | 4GB LPDDR4 |
| 存储 | microSD |
| 散热 | 80°C 触发降频 |

平台限制直接影响配置选择：SLAM 栈内存限制为 16MB、AMCL 粒子数上限 1000、EKF 频率 20Hz、全局 costmap 更新频率 2Hz。

## 当前硬件

| 设备 | 型号 | 接口 |
|------|------|------|
| 激光雷达 | YDLIDAR X2 | USB-UART (`/dev/ydlidar`) |
| 底盘控制板 | STM32F407 | USB-UART (`/dev/stm32_chassis`) |
| IMU | MPU6050 | 板载 I2C（STM32 侧） |
| 电机 | JGB37-520 × 2 | TB67H450FNG 驱动 |
| 编码器 | AB 相霍尔 × 2 | TIM 编码器模式 |

## 明确排除项

以下设备不属于当前硬件，相关代码路径已移除或禁用：

- **Astra Pro 深度摄像头**：属于后续 AI 扩展（Phase 7），当前激光 SLAM 主链路不包含摄像头
- **四轮差速 / 麦轮底盘**：旧 x86 工程遗留，当前为两轮差速
- **ESP01S 通信**：预留路径（USART2），不参与 ROS2 主链路

## 核心任务

1. 接入 YDLIDAR X2，发布可靠 `/scan`
2. 接入 STM32 底盘串口协议（v1），发布 `/odom`、`/tf`、`/imu/data`
3. 支持 SLAM Toolbox 建图（手动 + 自主探索）
4. 支持 Nav2 定位导航（AMCL + DWB Planner）
5. 支持手动遥控和键盘控制
6. 后续接入摄像头实现 AI 检测识别

## 与下位机的同步约定

本工程与下位机工程 [YQuark/SlamRobot_Chassis_Control_V1.0](https://github.com/YQuark/SlamRobot_Chassis_Control_V1.0) 同步推进。以下内容以下位机代码为权威来源：

- 串口帧格式、命令字、校验算法
- 数据字段顺序、类型、缩放系数
- 控制模式枚举、错误位定义
- 运动学参数（轮径、轮距、编码器 counts/rev）
- IMU 坐标系、电压电流换算参数
