# 上位机与下位机代码二次审计执行计划

## 内部执行等级

L：本轮是跨代码树静态审计和文档更新，适合单通道串行执行。

## 执行步骤

1. 重新确认 WPS 与工作区状态。
2. 清点上位机自研目录、集成目录和下位机工程文件。
3. 对比本轮关键改动：
   - `bridge_node.py` 速度上限、状态扩展、odom stale 策略、IMU 发布。
   - `system.launch.py`、`base.launch.py`、`robot.sh` 的默认参数、EKF、串口探测和雷达串口排除。
   - STM32 `robot_control.c`、`link_proto.c` 的运动学、模式切换和状态上报。
   - ESP01S Web 控制、状态 JSON、Wi-Fi 配置接口。
   - YDLidar scan 发布与 invalid range 语义。
4. 形成二次审计结论：已改善项、新增风险、残留风险、修复优先级。
5. 更新既有 Word 审计说明。
6. 输出清理回执，明确远程验证要求。

## 验证策略

本地仅进行静态文件阅读、差异查看和证据行定位。运行验证需在远程机器人环境执行。
