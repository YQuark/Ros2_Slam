# 导航拆解调试

本文档用于诊断导航链路异常时的分步排查。日常使用优先执行 `./robot.sh navigation ...`。

## 0. 底盘控制验证

仅启动 `stm32_bridge`，不启动雷达、AMCL、Nav2、RViz：

```bash
./robot.sh base-test --rotate-only
```

期望现象：

- 车原地左转一小段，再原地右转一小段
- 日志中连续出现 `cmd_vel_rx`、`drive_tx`、`status_summary`
- `status_summary` 中的 `odom` 和 `wheels` 在运动时不应一直为 0

原地转向稳定后，测试直线：

```bash
./robot.sh base-test --with-linear
```

### 角速度标定

量化命令角速度和实际角速度的差异：

```bash
./robot.sh base-test --calibrate-angular
```

自定义扫描档位：

```bash
BASE_TEST_ANGULAR_SERIES=0.5,1.0,1.5,2.0,2.5 ./robot.sh base-test --calibrate-angular
```

`Cmd log` 中输出 `CALIBRATION ...` 行，关键字段：

- `cmd_wz_deg_s` — 发给底盘的角速度命令
- `effective_wz_deg_s` — 根据 `/odom` 计算的实际平均角速度
- `response_ratio` — 实际角速度 / 命令角速度

标定结果可直接判断：

- 底盘从哪个角速度开始克服静摩擦
- 左转和右转是否不对称
- 导航参数中的 `min_speed_theta` 应设到什么量级

如果小角速度几乎不动、1.0~2.0 rad/s 才进入可用区、左右转不对称，调参策略：

- 抬高 `min_speed_theta`，避开无效小转向
- 放宽 `yaw_goal_tolerance`，减少终点附近反复小角修正
- 将 1.0~1.8 rad/s 作为主要工作区

方向反了应先修复底盘层线速度、角速度方向或里程计符号，而非修改 Nav2 参数。`status=NO_ODOM` 说明测试阶段未收到有效 `/odom`。

## 1. 里程计和 TF

底盘能响应 `/cmd_vel` 后，验证 `/odom` 和 TF：

```bash
ros2 topic echo /odom --once
ros2 run tf2_ros tf2_echo odom base_link
```

期望：

- 车前进时 `odom.pose.pose.position.x` 增加
- 原地左转时 yaw 方向与实际一致
- `odom → base_link` 持续可查

## 2. 雷达和外参

确认雷达点云与车体方向一致：

```bash
./robot.sh sensor lidar
```

另一终端：

```bash
ros2 topic hz /scan
ros2 run tf2_ros tf2_echo base_link laser_frame
```

期望：

- `/scan` 频率稳定
- RViz 中车前方障碍物显示在 `base_link` 前方
- `Fixed Frame=base_link` 且只看 `LaserScan` 时，左侧障碍物显示在左侧

左右镜像处理顺序：

1. 检查 `src/robot_bringup/config/ydlidar_x2.yaml` 中的 `inverted`
2. 测试 `--lidar-inverted` / `--no-lidar-inverted`
3. 测试 `--lidar-reversion` 或 `--lidar-yaw-*`
4. 错误手性下建出的旧地图直接废弃

## 3. 建图和保存

底盘、里程计、雷达都稳定后再建图：

```bash
./robot.sh mapping lidar --real-base --ekf-base
MAP_PADDING_METERS=1.5 ./robot.sh save-map my_map
```

## 4. AMCL 定位

标准导航流程异常时，单独拉起定位链：

```bash
./robot.sh navigation /home/robot/ros2_maps/my_map.yaml --real-base --ekf-base --localization-only
```

RViz 中使用 `2D Pose Estimate` 设置当前位置。期望激光点与地图墙面重合。

## 5. 全局规划

AMCL 对齐后再开 Nav2：

```bash
./robot.sh navigation --nav2-only
```

`--nav2-only` 启动前检查 `/map_server`、`/amcl` 是否 active，并等待 `/amcl_pose`。

RViz 中用 `2D Goal Pose` 点目标，先只看 `/plan` 是否生成。

## 6. 局部控制

`/plan` 有路线但车不按路线走时：

```bash
ros2 topic hz /cmd_vel
ros2 topic echo /cmd_vel
ros2 topic echo /local_plan
```

判断 Nav2 是否持续给底盘发速度，以及局部轨迹是否合理。

## 7. Nav2 关键参数

参数文件：`src/robot_bringup/config/nav2_params_robot.yaml`

### 控制器参数（DWB Planner）

| 参数 | 当前值 | 说明 |
|------|--------|------|
| `controller_frequency` | 15.0 Hz | 局部控制频率 |
| `max_vel_x` | 0.26 m/s | 最大线速度 |
| `max_vel_theta` | 1.00 rad/s | 最大角速度 |
| `min_speed_theta` | 0.40 rad/s | 最小有效角速度，低于此值视为无效 |
| `yaw_goal_tolerance` | 0.35 rad | 航向目标容差 |

### 代价地图参数

| 参数 | 当前值 | 说明 |
|------|--------|------|
| `footprint` | `[[0.11, 0.075], [0.11, -0.075], [-0.11, -0.075], [-0.11, 0.075]]` | 车体外廓（22cm × 15cm） |
| `inflation_radius` | 0.25m | 障碍物膨胀半径 |
| `planner_frequency` | 5.0 Hz (全局) | 全局规划频率 |

### 行为树结构

默认导航 BT：`navigate_to_pose_recovery.xml`

```text
RecoveryNode(retries=6) "NavigateRecovery"
├── PipelineSequence "NavigateWithReplanning"
│   ├── RateController(1Hz)
│   │   └── RecoveryNode(retries=1) "ComputePathToPose"
│   │       ├── ComputePathToPose(planner_id=GridBased)
│   │       └── ClearEntireCostmap(global)
│   └── RecoveryNode(retries=1) "FollowPath"
│       ├── FollowPath(controller_id=FollowPath)
│       └── ClearEntireCostmap(local)
└── ReactiveFallback "NavigateRecoveryFallback"
    ├── GoalUpdated()
    └── RoundRobin "NavigateRecoveryActions"
        ├── ClearCostmaps(local + global)
        ├── Spin(1.57 rad)
        ├── Wait(4.0 s)
        └── BackUp(0.20 m, 0.05 m/s)
```

行为树节点含义：

| 节点 | 功能 |
|------|------|
| `ComputePathToPose` | 全局路径规划 |
| `FollowPath` | 局部路径跟踪 |
| `ClearEntireCostmap` | 清空代价地图 |
| `Spin` | 原地旋转（用于清除局部障碍） |
| `BackUp` | 低速倒车脱困 |
| `Wait` | 等待，让传感器数据更新 |
| `GoalUpdated` | 检测是否有新目标 |
| `RateController` | 控制子树执行频率 |

### 恢复行为流程

当路径跟踪失败时，行为树执行以下恢复序列（RoundRobin 轮询）：

1. 清空局部和全局代价地图
2. 原地旋转 90 度（1.57 rad）
3. 等待 4 秒
4. 低速倒车 0.20 m

每种恢复动作执行后重新尝试规划。6 次重试全部失败后放弃导航。

## 8. 面板导航

前面步骤都稳定后再做面板按钮或网页点选。面板本质上只需发布 `NavigateToPose` action 或 `/goal_pose`。
