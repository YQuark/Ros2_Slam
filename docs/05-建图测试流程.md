# 建图测试流程

本文档说明当怀疑底盘里程计、EKF 或雷达外参影响建图质量时，如何进行可复现的 A/B 测试。

## 1. 测试前提

- 雷达健康检查通过（`./robot.sh check lidar`）
- 底盘可正常受控（`./robot.sh check chassis`）
- 使用统一入口 `robot.sh`
- 建议先从默认 `mapping` 档开始，不直接测 `precision`

## 2. 测试目标

一次测试最多回答一个主问题。常见问题：

1. `--ekf-base` 是正收益还是负收益
2. 当前问题更像底盘航向误差，还是雷达外参误差
3. 里程计缩放系数是否准确

同时修改多个参数（EKF、SLAM 档位、雷达外参）会导致无法判断哪个因素起作用。

> **已废弃参数**：`BASE_USE_STATUS_YAW`、`BASE_STATUS_YAW_MODE`、`BASE_STATUS_YAW_JUMP_REJECT_DEG`。这些参数对应旧协议的 `yaw_est` 字段，当前 v1 协议（`0xA5 0x5A` 帧格式）不包含该字段，相关参数已移除。

## 3. 推荐测试顺序

### 第一轮：验证 EKF 影响

```bash
# 基线组：不启用 EKF
./robot.sh mapping lidar --real-base

# EKF 组：启用 EKF
./robot.sh mapping lidar --real-base --ekf-base
```

这轮只回答：当前 EKF 配置是在帮忙，还是在把同源数据又滤坏一遍。

### 第二轮：验证里程计缩放

如果第一轮发现直线行走有明显偏移，检查里程计缩放系数：

```bash
# 通过环境变量临时调整线速度缩放
BASE_ODOM_LINEAR_SCALE=1.05 ./robot.sh mapping lidar --real-base

# 或角速度缩放
BASE_ODOM_ANGULAR_SCALE=0.95 ./robot.sh mapping lidar --real-base
```

### 第三轮：精细建图

前两轮稳定后再测精细档：

```bash
./robot.sh mapping lidar precision --real-base
./robot.sh mapping lidar precision --real-base --ekf-base
```

## 4. 标准测试路线

每组测试走完全相同的路线：

1. 原地静止 10 秒
2. 直线前进 2 米
3. 直线后退 2 米
4. 原地左转 90 度，停 2 秒
5. 再左转 90 度
6. 贴墙走矩形一圈
7. 回到起点附近静止 5 秒

场地有限时至少保留：静止、直线、两次 90 度转向、一圈矩形。

## 5. 录包方式

终端 1 启动系统，终端 2 录包：

```bash
# 基线组
cd /home/robot/ros2_ws/launch_scripts
./robot.sh mapping lidar --real-base

# 另一终端
./record_mapping_test.sh baseline --notes "./robot.sh mapping lidar --real-base"
```

`record_mapping_test.sh` 自动创建带时间戳的测试目录，生成 `TEST_RECORD.md` 和 `metadata.json`，录制以下话题：

- `/scan`、`/odom`、`/imu/data`、`/tf`、`/tf_static`、`/odometry/filtered`

注意：未启用 `--ekf-base` 时，`/odometry/filtered` 无数据属于正常。

## 6. 实时量化监视

第三个终端运行实时指标监视：

```bash
./monitor_mapping_metrics.py --csv-out ~/ros2_test_records/live_metrics.csv
```

输出量：

- `/odom` 的 `x y yaw vx wz`
- `/imu/data` 的 `yaw gz ax ay az`
- `/odometry/filtered` 的 `x y yaw vx wz`
- `odom - imu` 的航向差
- 静止段的位移漂移和航向漂移

静止 10 秒期间若 `static.odom_drift` 或 `static.imu_yaw_drift` 持续增大，说明底盘数据流本身存在漂移。

## 7. 每组重点观察指标

- 静止时地图或 `/odom` 是否自行漂移
- 直线时车是否向一侧偏
- 两次 90 度转向后角点是否接近直角
- 矩形四条边是否平直
- 回到起点后地图是否基本对齐

补充观察：

- `/odom` 和实际车体方向是否一致
- 转弯时地图是否出现"边走边扭"
- 停车后地图是否继续缓慢转动

## 8. 记录规范

每组测试结束后在 `TEST_RECORD.md` 中填写：

- 实际启动命令
- 静止漂移是否存在
- 直线横漂是否明显
- 90 度角是否正常
- 矩形是否闭合
- 与上一组相比更好、持平还是更差

记录粒度示例：

- 静止 10 秒，地图基本不转
- 直线 2 米，右偏约 8cm
- 两次 90 度后，墙角有轻微钝化
- 回到起点后，闭环误差约 15cm

## 9. 结果判断

### 情况 A：基线差，EKF 变差

- 结论：当前 EKF 配置是负收益
- 处理：建图链路暂不启用 EKF

### 情况 B：基线差，EKF 变好

- 结论：EKF 融合有效
- 处理：继续使用 `--ekf-base`

### 情况 C：各组都差不多歪

- 结论：更像雷达外参或底盘几何标定问题
- 处理：检查 `base_link → laser_frame`、轮距、速度缩放

### 情况 D：静止时都自行漂

- 结论：底盘里程计本身不稳定
- 处理：检查轮速精度、编码器分辨率、IMU 角速度偏置

## 10. 最小验证集

只做两组即可获得最有价值的对比：

```bash
./robot.sh mapping lidar --real-base
./robot.sh mapping lidar --real-base --ekf-base
```

## 11. 参数调优顺序

A/B 结果明确后，按以下顺序逐步调整：

1. 先定里程计缩放系数（`odom_linear_scale`、`odom_angular_scale`）
2. 再定是否保留 EKF
3. 再校雷达外参（`lidar_tf_*`）
4. 最后调 SLAM Toolbox 参数档

顺序颠倒会导致测试结果失真。
