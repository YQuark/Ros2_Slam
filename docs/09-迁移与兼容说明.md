# 迁移与兼容说明

这份文档用于说明旧入口和新入口的关系。

## 1. 当前推荐入口

统一推荐入口是:

```bash
cd /home/robot/ros2_ws/launch_scripts
./robot.sh <command>
```

其中最常用的是:

- `mapping`
- `navigation`
- `save-map`
- `doctor`
- `check lidar`
- `stop`

## 2. 旧脚本的定位

仓库里仍保留了一些旧名字脚本，它们现在主要承担兼容壳作用。
它们的目标不是再维护一套独立逻辑，而是把用户调用转到统一入口。

## 3. 为什么要迁移

旧体系的问题主要有:

- 入口太多
- 参数在多个脚本里重复
- 文档和入口不一致
- 同一能力有多个平行实现

统一到 `robot.sh` 和 `system.launch.py` 后，维护成本会明显下降。

## 4. 迁移原则

- 新文档只写新入口
- 旧命令如果还存在，只作为兼容说明
- 不再新增与统一入口平行的脚本
- 新功能优先补到统一入口和统一 launch

## 5. 迁移检查点

如果你改过工程，至少确认这几件事:

- `robot.sh` 还能直接启动建图和导航
- `system.launch.py` 的参数仍然对得上
- 设备自动识别还能工作
- 雷达健康检查还能工作
- RViz 配置仍然能打开
- 根目录 `README.md` 和 `docs/` 的入口说明仍然一致
