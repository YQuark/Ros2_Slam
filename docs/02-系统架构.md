# 系统架构

本文档说明工程分层结构、ROS2 节点组成、话题映射、TF 树和关键数据流。

## 1. 分层

工程分五层：

| 层级 | 职责 | 关键文件 |
|------|------|----------|
| 运维入口 | 将用户意图翻译为标准命令 | `launch_scripts/robot.sh` |
| 编排入口 | 根据模式参数组装节点 | `src/robot_bringup/launch/system.launch.py` |
| 设备接入层 | 传感器驱动、底盘串口桥接 | `ydlidar_ros2_driver`、`stm32_robot_bridge` |
| 算法层 | SLAM、定位、路径规划 | `slam_toolbox`、`Nav2`、`robot_localization` |
| 参数与可视化 | 配置文件、RViz 配置 | `robot_bringup/config`、`robot_bringup/rviz` |

## 2. 仓库目录

ros2_ws/

## 3. 模式矩阵

`system.launch.py` 通过以下参数组合确定启动哪些节点：

| 参数 | 取值 | 说明 |
|------|------|------|
| `mode` | `mapping` / `navigation` | 主模式 |
| `mapping_source` | `lidar` | 建图传感器（仅雷达） |
| `base_mode` | `real` / `fake` / `none` | 底盘模式 |
| `base_fusion_mode` | `none` / `ekf` | 是否启用 EKF 融合 |

建图模式组合：

- `mapping` + `lidar` + `real` — 真实底盘雷达建图
- `mapping` + `lidar` + `fake` — 虚拟底盘雷达建图（仅验证雷达和 SLAM）
- `mapping` + `lidar` + `none` — 无底盘建图（需 `fallback_static_odom:=true`）

导航模式组合：

- `navigation` + `real` — 真实底盘导航
- `navigation` + `fake` — 虚拟底盘导航（仿真测试）

## 4. ROS2 节点列表

### 传感器层（sensors.launch.py）

| 节点 | 包 | 可执行文件 | 职责 |
|------|-----|-----------|------|
| `ydlidar_ros2_driver_node` | `ydlidar_ros2_driver` | `ydlidar_ros2_driver_node` | 发布 `/scan_raw` |
| `scan_normalizer` | `robot_bringup` | `scan_normalizer.py` | `/scan_raw → /scan`，裁剪点数和角度范围 |
| `static_tf_pub_laser` | `tf2_ros` | `static_transform_publisher` | `base_link → laser_frame` |

### 底盘层（base.launch.py）

| 节点 | 包 | 可执行文件 | 职责 |
|------|-----|-----------|------|
| `stm32_bridge` | `stm32_robot_bridge` | `bridge_node` | 串口桥接，发布 `/odom`、`/tf`、辅助话题 |

### 静态 TF（system.launch.py）

| 节点 | TF | 说明 |
|------|-----|------|
| `static_tf_base_footprint` | `base_link → base_footprint` | 地面投影，z=0 |
| `static_tf_imu_link` | `base_link → imu_link` | IMU 安装偏移 |

### EKF 融合层（base_ekf.launch.py，`base_fusion_mode:=ekf`）

| 节点 | 包 | 可执行文件 | 职责 |
|------|-----|-----------|------|
| `ekf_node` | `robot_localization` | `ekf_node` | 融合 `/odom` + `/imu/data`，发布 `/odometry/filtered` 和 `odom → base_link` |

### 建图模式（mapping）

| 节点 | 包 | 可执行文件 | 职责 |
|------|-----|-----------|------|
| `async_slam_toolbox_node` | `slam_toolbox` | `async_slam_toolbox_node` | SLAM 建图，发布 `/map` |
| `map_to_pointcloud_viz` | `robot_bringup` | `map_to_pointcloud_viz.py` | `/map → /map_points`，RViz 可视化辅助 |
| `frontier_explorer` | `robot_bringup` | `frontier_explorer.py` | 自主探索建图（`auto_mapping_drive:=true` 时） |

### 导航模式（navigation）

| 节点 | 包 | 职责 |
|------|-----|------|
| `map_server` | `nav2_map_server` | 加载地图文件 |
| `amcl` | `nav2_amcl` | 粒子滤波定位，发布 `map → odom` TF |
| `controller_server` | `nav2_controller` | 局部路径跟踪（DWB Planner） |
| `planner_server` | `nav2_planner` | 全局路径规划（NavFn） |
| `bt_navigator` | `nav2_bt_navigator` | 行为树驱动导航流程 |
| `waypoint_follower` | `nav2_waypoint_follower` | 路点跟踪 |
| `recoveries_server` | `nav2_recoveries` | 恢复行为（旋转、清空代价地图） |
| `lifecycle_manager_*` | `nav2_lifecycle_manager` | 节点生命周期管理 |

| 话题 | 消息类型 | 订阅者 | 说明 |
|------|----------|--------|------|
| `/cmd_vel` | `geometry_msgs/Twist` | `stm32_bridge` | 速度指令 |

## 6. TF 树

```text
map ──→ odom ──→ base_link ──→ base_footprint
                     │
                     ├──→ laser_frame
                     └──→ imu_link
```

各 TF 的发布者：

| TF | 发布者 | 条件 |
|----|--------|------|
| `map → odom` | `amcl` | 导航模式 |
| `odom → base_link` | `stm32_bridge` | `publish_tf:=true`（默认） |
| `odom → base_link` | `ekf_node` | `base_fusion_mode:=ekf` 时替代上者 |
| `base_link → laser_frame` | `static_tf_pub_laser` | 始终 |
| `base_link → base_footprint` | `static_tf_base_footprint` | 始终 |
| `base_link → imu_link` | `static_tf_imu_link` | 始终 |

当 `base_fusion_mode:=ekf` 时，`system.launch.py` 自动设置 `publish_tf:=false`，由 EKF 节点发布 `odom → base_link`。

## 7. 关键数据流

### 雷达建图

```text
ydlidar_ros2_driver_node → /scan_raw → scan_normalizer → /scan → slam_toolbox → /map
```

### 真实底盘导航

```text
Nav2(bt_navigator)
  → /cmd_vel → stm32_bridge → STM32 下位机
  ← STATUS 帧 ← stm32_bridge → /odom
  ← STATUS 帧 ← stm32_bridge → /imu/data
  /odom + /imu/data → ekf_node → /odometry/filtered + odom→base_link
  /scan → amcl → map→odom
```

### 底盘串口桥接内部

```text
/cmd_vel → STM32Bridge.on_cmd_vel()
  → control_loop() [20Hz]
    → encode_drive_payload(vx, wz)
    → write_frame(CMD_SET_VELOCITY, payload) → 串口
  → status_poll() [20Hz]
    → write_frame(CMD_STATUS, b'') → 串口
    ← read_serial_frames() → handle_frame()
      → 解析 47 字节 STATUS
      → 计算轮速里程计 → publish_odom()
      → 发布 /chassis/status, /battery_state, /motor/*_current
```

## 8. Nav2 子系统

导航模式启动以下 Nav2 组件：

| 组件 | 职责 | 关键参数文件 |
|------|------|-------------|
| AMCL | 粒子滤波定位 | `nav2_params_robot.yaml` 中 `amcl:` 段 |
| DWB Planner | 局部路径跟踪 | `nav2_params_robot.yaml` 中 `controller_server:` 段 |
| NavFn Planner | 全局路径规划 | `nav2_params_robot.yaml` 中 `planner_server:` 段 |
| Costmap | 全局/局部代价地图 | `nav2_params_robot.yaml` 中 `global_costmap:` / `local_costmap:` 段 |
| BT Navigator | 行为树导航 | `navigate_to_pose_recovery.xml` |
| Recoveries | 旋转恢复、清空代价地图 | `nav2_params_robot.yaml` 中 `recoveries_server:` 段 |

行为树文件位于 `src/robot_bringup/behavior_trees/`：

- `navigate_to_pose_recovery.xml` — 默认导航 BT，包含重规划、清空代价地图和旋转恢复
- `mapping_navigate_to_pose.xml` — 自主探索建图用 BT

## 9. SLAM Toolbox 参数档

建图模式通过 `--profile` 参数选择 SLAM 参数档：

| 档位 | 参数文件 | 说明 |
|------|----------|------|
| `mapping`（默认） | `slam_toolbox_mapping.yaml` | 标准建图 |
| `fast` | `slam_toolbox_mapping_fast.yaml` | 快速建图，降低精度 |
| `realtime` | `slam_toolbox_mapping_realtime.yaml` | 实时模式 |
| `precision` | `slam_toolbox_mapping_lidar_precision.yaml` | 精细建图 |

选择逻辑在 `robot.sh` 的 `choose_slam_params()` 函数中实现。

## 10. 设计原则

- 用户入口统一为 `robot.sh`，工程编排统一为 `system.launch.py`
- 设备识别优先自动探测，不依赖硬编码串口路径
- 参数文件与启动入口解耦
- 第三方源码依赖收口在 `src/third_party/`
- 文档按任务组织，不按文件堆砌
