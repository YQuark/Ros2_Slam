# 系统架构

这份文档说明工程如何分层、文件放在哪里、各个入口之间如何衔接。

## 1. 分层

当前工程分五层:

1. 运维入口: `launch_scripts/robot.sh`
2. 编排入口: `src/robot_bringup/launch/system.launch.py`
3. 设备接入层: 雷达驱动、相机驱动、`stm32_robot_bridge`
4. 算法层: `slam_toolbox`、`Nav2`、`robot_localization`
5. 参数与可视化层: `robot_bringup/config`、`robot_bringup/rviz`

`robot.sh` 负责把用户意图翻译成标准命令。
`system.launch.py` 负责根据 `mode`、`mapping_source`、`base_mode`、`base_fusion_mode` 组装实际节点。

## 2. 仓库目录约定

```text
ros2_ws/
├── docs/                  # 主文档体系
├── launch_scripts/        # 运维 CLI、诊断脚本、兼容包装
├── src/
│   ├── robot_bringup      # 技术编排与参数总入口
│   ├── stm32_robot_bridge # 底盘串口桥接
│   ├── third_party/       # vendored 依赖
│   └── 其它驱动包
├── build/ install/ log/   # colcon 生成目录，不属于源码层
└── README.md              # 仓库根入口
```

## 3. 模式矩阵

### 建图

- `mode:=mapping`
- `mapping_source:=camera` 或 `lidar`
- `base_mode:=none|fake|real`
- `base_fusion_mode:=none|ekf`

### 导航

- `mode:=navigation`
- `base_mode:=fake|real`
- `base_fusion_mode:=none|ekf`
- 必须提供 `map_file`

## 4. 关键数据流

### 雷达建图

`ydlidar_ros2_driver_node` -> `/scan` -> `slam_toolbox` -> `/map`

### 摄像头建图

`astra_camera` -> `/camera/depth/image_raw` -> `depthimage_to_laserscan` -> `/camera/scan` -> `slam_toolbox`

### 导航

`Nav2` -> `/cmd_vel` -> `stm32_robot_bridge` -> 底盘下位机 -> 编码器/IMU反馈 -> `/odom` 和 `odom->base_link`

### 底盘 EKF 融合

`stm32_robot_bridge` -> `/odom` + `/imu/data` -> `robot_localization(ekf_node)` -> `/odometry/filtered` + `odom->base_link`

## 5. TF 关系

工程里最重要的 TF 关系是:

- `map -> odom`
- `odom -> base_link`
- `base_link -> laser_frame`

在真实底盘模式下：

- 未启用 EKF 时，`odom -> base_link` 由 `stm32_robot_bridge` 发布
- 启用 `base_fusion_mode:=ekf` 时，`odom -> base_link` 由 `robot_localization` 发布
- `base_link -> laser_frame` 由静态 TF 发布，当前默认值来自实装雷达外参

## 6. 统一入口参数

`src/robot_bringup/launch/system.launch.py` 的核心参数包括:

- `mode`
- `mapping_source`
- `base_mode`
- `base_fusion_mode`
- `use_camera`
- `use_lidar`
- `use_rviz`
- `map_file`
- `base_port`
- `lidar_params_file`
- `nav2_params_file`

更完整的参数默认值在 launch 文件里定义，当前文档只保留工程使用层的关键项。

## 7. 设计原则

- 用户只记 `robot.sh`
- 工程内部只认 `system.launch.py`
- 设备识别优先自动探测
- 参数文件和启动入口解耦
- vendored 依赖集中放到 `src/third_party/`
- 文档按任务组织，不按文件堆砌
