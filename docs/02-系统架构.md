# 系统架构

这份文档说明工程如何分层，以及各个入口之间的关系。

## 1. 分层

当前工程分三层:

1. 运维入口: `launch_scripts/robot.sh`
2. 编排入口: `src/robot_bringup/launch/system.launch.py`
3. 设备与算法节点: 雷达驱动、相机驱动、`slam_toolbox`、`Nav2`、`stm32_robot_bridge`

`robot.sh` 负责把用户意图翻译成标准命令。
`system.launch.py` 负责根据 `mode`、`mapping_source`、`base_mode` 组装实际节点。

## 2. 模式矩阵

### 建图

- `mode:=mapping`
- `mapping_source:=camera` 或 `lidar`
- `base_mode:=none|fake|real`

### 导航

- `mode:=navigation`
- `base_mode:=fake|real`
- 必须提供 `map_file`

## 3. 关键数据流

### 雷达建图

`ydlidar_ros2_driver_node` -> `/scan` -> `slam_toolbox` -> `/map`

### 摄像头建图

`astra_camera` -> `/camera/depth/image_raw` -> `depthimage_to_laserscan` -> `/camera/scan` -> `slam_toolbox`

### 导航

`Nav2` -> `/cmd_vel` -> `stm32_robot_bridge` -> 底盘下位机 -> 编码器/IMU反馈 -> `/odom` 和 `odom->base_link`

## 4. TF 关系

工程里最重要的 TF 关系是:

- `map -> odom`
- `odom -> base_link`
- `base_link -> laser`

在真实底盘模式下，`map -> odom` 由定位或 SLAM 修正，`odom -> base_link` 由底盘反馈闭环。

## 5. 统一入口参数

`src/robot_bringup/launch/system.launch.py` 的核心参数包括:

- `mode`
- `mapping_source`
- `base_mode`
- `use_camera`
- `use_lidar`
- `use_rviz`
- `map_file`
- `base_port`
- `lidar_params_file`
- `nav2_params_file`

更完整的参数默认值在 launch 文件里定义，当前文档只保留工程使用层的关键项。

## 6. 设计原则

- 用户只记 `robot.sh`
- 工程内部只认 `system.launch.py`
- 设备识别优先自动探测
- 参数文件和启动入口解耦
- 文档按任务组织，不按文件堆砌
