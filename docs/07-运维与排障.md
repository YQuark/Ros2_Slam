# 运维与排障

这份文档把日常检查和排障动作收口到一起。

## 1. 先记住当前正式入口

- 正式用户入口：`./robot.sh`
- 正式导航主流程：`./robot.sh navigation ...`
- 两阶段导航：只作为回退和诊断
- `start_*.sh`：只做兼容转发，不再作为主入口

## 2. 常用命令

```bash
cd /home/robot/ros2_ws/launch_scripts
./robot.sh doctor
./robot.sh stop
./robot.sh check lidar
./robot.sh check navigation
./robot.sh sensor lidar
./robot.sh base
```

## 3. 优先检查顺序

1. `./robot.sh doctor`
2. 设备接线和供电
3. 串口是否冲突或被旧进程占用
4. 雷达是否有 `/scan`
5. 底盘是否有 `/odom`
6. RViz 是否只是显示异常
7. 导航前是否已经完成 `2D Pose Estimate`

## 4. 导航异常时先怎么分流

如果 `./robot.sh navigation ...` 启动后异常，先判断是哪一类：

- 定位没收敛：先做 `2D Pose Estimate`
- 激光和地图不重合：先查激光手性和外参
- 点 `2D Goal Pose` 后完全没反应：先跑 `./robot.sh check navigation`
- 有 `/plan` 但车不走：查 `/cmd_vel`、`/local_plan` 和底盘
- 路径太保守或拐弯太生硬：先确认 footprint 是否和实物一致，再看 `/local_plan`
- 连 Nav2 都起不来：再回退到 `--localization-only` / `--nav2-only`

两阶段回退：

```bash
./robot.sh navigation --real-base --ekf-base --localization-only
./robot.sh navigation --nav2-only
```

## 5. 雷达常见问题

- `/scan` 数量为 0
- `Check Sum` 或 `Failed to get scan`
- 左右镜像

优先动作：

1. 换 USB 线和检查供电
2. 跑 `./robot.sh check lidar`
3. 用 `./robot.sh sensor lidar` 单独验证雷达
4. 再测试 `--lidar-inverted`、`--lidar-reversion`、`--lidar-yaw-*`

不要先从 odom 符号链去修激光左右镜像。

## 6. 底盘常见问题

- 串口识别成错误设备
- 底盘不响应 `/cmd_vel`
- `/odom` 不更新

优先动作：

1. 看 `base_mode`
2. 看 `base_port`
3. 看底盘是否上电
4. 看串口是否被占用
5. 跑 `./robot.sh base-test --rotate-only`

## 7. RViz 常见问题

如果地图层显示异常，先区分是渲染问题还是数据问题：

- `/map` 是否仍在发布
- `/scan` 是否仍在更新
- `slam_toolbox` 是否仍在运行

## 8. 话题和节点检查

```bash
ros2 topic list | grep -E '^/scan$|^/camera/scan$|^/map$|^/odom$|^/odometry/filtered$|^/imu/data$|^/cmd_vel$'
ros2 node list
```

如果节点存在但数据不动，问题通常在串口、传感器、参数或状态机前置条件。

导航不响应目标时，优先按下面顺序看：

```bash
./robot.sh check navigation
ros2 topic echo /goal_pose --once
ros2 topic echo /plan --once
ros2 topic echo /local_plan --once
ros2 topic hz /cmd_vel
```

当前默认导航已切到真实外廓 `22cm x 15cm`，并允许低速倒车。
如果出现贴障过近、挤角或擦碰风险，先确认 footprint 是否与实物一致，再考虑回调安全边界。

## 9. 当前默认设备约定

- 底盘默认串口：`/dev/ttyUSB1`
- 雷达默认串口：`/dev/ttyUSB0`
- 默认雷达参数：`src/robot_bringup/config/ydlidar_X2_mapping.yaml`
