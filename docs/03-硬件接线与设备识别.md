# 硬件接线与设备识别

本文档说明当前硬件清单、串口设备识别机制和常见问题排查。

## 1. 硬件清单

| 设备 | 型号 | 接口 | 设备名 |
|------|------|------|--------|
| 激光雷达 | YDLIDAR X2 | USB-UART (CP210x) | `/dev/ydlidar`（udev 规则）或 `/dev/ttyUSB*` |
| 底盘控制板 | STM32F407 | USB-UART (CP2102) | `/dev/stm32_chassis`（udev 规则）或自动探测 |
| IMU | MPU6050 | STM32 侧 I2C | 通过底盘 STATUS 帧上报 |
| 电机 | JGB37-520 × 2 | TB67H450FNG 驱动 | — |

不属于当前硬件的设备：

- Astra Pro 深度摄像头 — 属于后续 AI 扩展（Phase 7），当前激光 SLAM 主链路不包含摄像头

## 2. udev 规则配置

`launch_scripts/setup_lidar_permissions.sh` 完成以下配置：

1. 将 `robot` 用户加入 `dialout` 组（串口访问权限）
2. 写入 udev 规则 `/etc/udev/rules.d/99-ydlidar.rules`，为 CP210x 芯片创建稳定设备名 `/dev/ydlidar`

udev 规则内容：

```text
SUBSYSTEM=="tty", ATTRS{idVendor}=="10c4", ATTRS{idProduct}=="ea60", SYMLINK+="ydlidar", GROUP="dialout", MODE="0660"
```

执行方式：

```bash
sudo /home/robot/ros2_ws/launch_scripts/setup_lidar_permissions.sh
```

执行后需重新登录或重启，使 `dialout` 组成员变更生效。

## 3. 串口权限问题排查

当雷达驱动报 `Error, cannot bind to the specified [serial port]` 时，按以下步骤排查：

```bash
# 检查设备是否存在
ls -la /dev/ttyUSB* /dev/ydlidar 2>/dev/null

# 检查设备权限和所属组
ls -la /dev/ttyUSB0
# 期望: crw-rw---- 1 root dialout

# 检查当前用户组
groups
# 期望包含 dialout

# 如果不在 dialout 组
sudo usermod -aG dialout robot
# 重新登录后生效
```

临时测试（重启后失效）：

```bash
sudo chmod 666 /dev/ttyUSB0
```

## 4. 自动探测机制

### 底盘串口探测

`stm32_robot_bridge` 的 `bridge_node.py` 实现以下探测流程（`port:=auto` 时）：

1. `detect_cp210x_ports()` 扫描 `/dev/serial/by-id/` 和 `/dev/ttyUSB*`，通过以下特征识别 CP210x 芯片：
   - `by-id` 路径包含 `CP210`、`cp210`、`Silicon_Labs`、`USB*UART`
   - `/sys/class/tty/*/driver` 包含 `cp210x`
   - USB vendor ID 为 `10c4`，product ID 为 `ea60`
2. 排除 `excluded_ports` 中的设备（避免与雷达串口冲突）
3. 对每个候选端口执行 `probe_port_for_status()`：
   - 打开串口，发送 CMD_STATUS 请求
   - 验证收到 47 字节 STATUS 响应
   - 重复 3 次探测
4. 使用第一个通过探测的端口

Shell 层的 `detect_base_port.sh` 脚本提供类似的探测能力，用于 `robot.sh` 在启动前确定底盘端口。

### 雷达串口探测

`detect_lidar_port.sh` 在现有串口中排除已知底盘端口，将剩余设备用于雷达。

### 排除机制

`system.launch.py` 在启动底盘桥接时，自动将雷达参数文件中配置的 `port` 值传入 `excluded_ports`，防止底盘探测误用雷达串口。

## 6. 常见问题

| 问题 | 原因 | 解决方式 |
|------|------|----------|
| 串口打开失败 | 用户不在 `dialout` 组 | `sudo usermod -aG dialout robot` 后重新登录 |
| 设备名变化 | USB 口位不固定 | 使用 `port:=auto` 或 udev 规则创建稳定设备名 |
| 底盘和雷达串口互换 | 自动探测顺序不确定 | 配置 `excluded_ports` 或使用 udev 规则区分 |
| 串口被旧进程占用 | 上次运行未正常退出 | `./robot.sh stop` 后重试 |
| USB 线材不稳定 | 接触不良或线缆质量 | 更换 USB 线缆 |
