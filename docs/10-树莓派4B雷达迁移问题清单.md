本文记录从旧 x86/Ubuntu 20.04 文件系统迁移到当前树莓派 4B 环境，并尝试启动 YDLIDAR X2 雷达驱动时已经暴露的问题。

- 当前 ROS 版本：ROS 2 Humble

## 当前阻塞问题

### 1. robot 用户没有雷达串口权限

现象：

```text
Error, cannot bind to the specified [serial port:/dev/ttyUSB0] and [baudrate:115200]
Error initializing YDLIDAR check Comms.
```

进一步检查：

```text
设备权限: crw-rw---- 1 root dialout ... /dev/ttyUSB0
当前用户组: robot adm cdrom sudo dip plugdev lpadmin lxd sambashare
```

原因：

`/dev/ttyUSB0` 属于 `root:dialout`，当前 `robot` 用户不在 `dialout` 组，因此 YDLidar SDK 无法打开串口。

状态：

- 已在脚本中加入权限预检和清晰错误提示。
- 仍需要在树莓派本机输入 sudo 密码完成系统权限配置。

处理命令：

```bash
sudo /home/robot/ros2_ws/launch_scripts/setup_lidar_permissions.sh
```

临时测试命令：

```bash
sudo chmod 666 /dev/ttyUSB0
```

注意：执行 `usermod -aG dialout robot` 后需要重新登录或重启。

## 已发现并已处理的问题

### 2. 雷达默认设备名 `/dev/ydlidar` 不存在

现象：

配置文件默认使用：

```yaml
port: /dev/ydlidar
```

但当前系统实际只有 `/dev/ttyUSB0`，没有 `/dev/ydlidar`。

处理：

- `robot.sh` 已增加自动探测逻辑：如果 `/dev/ydlidar` 不存在，会调用 `detect_lidar_port.sh` 查找实际 `/dev/ttyUSB*`。
- 新增 `setup_lidar_permissions.sh`，写入 udev 规则，长期建立稳定设备名 `/dev/ydlidar`。

涉及文件：

- `/home/robot/ros2_ws/launch_scripts/robot.sh`
- `/home/robot/ros2_ws/launch_scripts/setup_lidar_permissions.sh`

### 3. 雷达健康检查缺少串口权限预检

现象：

原先 `check_lidar_health.sh` 会直接启动驱动，权限不足时只看到 SDK 的 `cannot bind` 错误，不容易定位到 Linux 用户组问题。

处理：

已在 `check_lidar_health.sh` 中加入：

- 串口是否存在
- 当前用户是否可读写串口
- 当前用户组
- 设备权限
- 修复命令提示

涉及文件：

- `/home/robot/ros2_ws/launch_scripts/check_lidar_health.sh`

### 4. `robot.sh mapping` 默认入口不适合当前雷达建图优先目标

现象：

普通命令环境中查询 `/dev/ttyUSB*` 为空，但提权查看主机设备时能看到 `/dev/ttyUSB0`。

原因：

处理：

涉及真实串口、`fuser`、`stty`、YDLidar 驱动启动的操作，需要在主机权限下执行。

### 6. `robot_bringup/scripts` 中部分 ROS 节点缺少可执行权限

现象：

雷达建图 launch 进入节点启动阶段后报错：

```text
executable 'scan_normalizer.py' not found on the libexec directory
'/home/robot/ros2_ws/install/robot_bringup/lib/robot_bringup'
```

原因：

`install/robot_bringup/lib/robot_bringup/scan_normalizer.py` 是指向源码目录的软链接。源码文件本身缺少可执行位时，ROS 2 launch 会认为该 executable 不可用。

处理：

已给 `src/robot_bringup/scripts/*.py` 补齐可执行权限。

验证：

短启动雷达建图时，以下节点已能正常拉起：

- `ydlidar_ros2_driver_node`
- `scan_normalizer.py`
- `static_transform_publisher`
- `fake_base_odom.py`
- `async_slam_toolbox_node`

### 8. SLAM Toolbox 最小激光距离参数触发边界警告

现象：

SLAM 启动时出现：

```text
minimum laser range setting (0.0 m) exceeds the capabilities of the used Lidar (0.1 m)
```

原因：

本地 SLAM 参数未显式设置 `min_laser_range`，使用了默认 `0.0`。设置为 `0.1` 时，slam_toolbox 对等于雷达下限的情况仍会提示边界警告。

处理：

已将各个雷达建图 SLAM 参数档统一设置为：

```yaml
min_laser_range: 0.09
```

涉及文件：

- `src/robot_bringup/config/slam_toolbox_mapping.yaml`
- `src/robot_bringup/config/slam_toolbox_mapping_fast.yaml`
- `src/robot_bringup/config/slam_toolbox_mapping_realtime.yaml`
- `src/robot_bringup/config/slam_toolbox_mapping_lidar_precision.yaml`
