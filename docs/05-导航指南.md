# 导航指南

导航流程的前提是你已经有一份可用地图。

## 1. 启动方式

真实底盘导航:

```bash
cd /home/robot/ros2_ws/launch_scripts
./robot.sh navigation /home/robot/ros2_maps/my_map.yaml --real-base
```

虚拟底盘导航联调:

```bash
./robot.sh navigation /home/robot/ros2_maps/my_map.yaml --fake-base
```

如果要临时禁用 RViz:

```bash
./robot.sh navigation /home/robot/ros2_maps/my_map.yaml --real-base --no-rviz
```

## 2. 导航链路

导航模式会启动:

- 定位
- `Nav2`
- 底盘桥接
- 可视化

目标控制链路是:

`Nav2` -> `/cmd_vel` -> `stm32_robot_bridge` -> 底盘

反馈链路是:

底盘 -> 编码器/IMU -> `/odom` -> `odom->base_link`

## 3. 你应该关注的现象

- RViz 中机器人位置会更新
- `/cmd_vel` 会有控制输出
- `/odom` 会随底盘反馈变化
- `map -> odom` 会在定位修正时更新

## 4. 何时用 fake base

`fake` 适合以下场景:

- 底盘还没接
- 只想先验证 `Nav2` 配置
- 想检查地图加载、目标点发布和 RViz 配置

## 5. 何时必须用 real base

`real` 适合以下场景:

- 要验证真实闭环
- 要看底盘能否跟随目标运动
- 要检查桥接和下位机反馈

## 6. 导航失败时优先看什么

- 地图文件是否存在
- 雷达是否正常出 `/scan`
- 底盘是否接入
- `base_mode` 是否选择正确
- 串口是否被占用
