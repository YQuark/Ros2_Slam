# 快速开始

本文档说明如何按照当前代码的正式流程启动系统。用户侧唯一推荐入口为 `launch_scripts/robot.sh`，内部编排入口 `system.launch.py` 不作为常规使用方式。

## 1. 环境要求

| 项目 | 要求 |
|------|------|
| 操作系统 | Ubuntu 22.04 LTS (arm64) |
| ROS2 版本 | Humble |
| 工作空间 | `/home/robot/ros2_ws` |
| 传感器 | YDLIDAR X2 已连接、STM32 底盘已上电 |
| 用户组 | `robot` 用户已加入 `dialout` 组 |

## 2. 构建

```bash
cd /home/robot/ros2_ws
source /opt/ros/humble/setup.bash
colcon build --symlink-install
source /home/robot/ros2_ws/install/setup.bash
```

## 3. robot.sh 命令列表

`robot.sh` 支持以下子命令：

| 子命令 | 功能 | 示例 |
|--------|------|------|
| `mapping` | 启动建图 | `./robot.sh mapping lidar --real-base` |
| `navigation` | 启动导航 | `./robot.sh navigation --real-base --ekf-base` |
| `sensor` | 启动单一传感器 | `./robot.sh sensor lidar` |
| `base` | 启动底盘桥接 | `./robot.sh base` |
| `base-test` | 测试底盘控制 | `./robot.sh base-test --rotate-only` |
| `full` | 启动全量系统 | `./robot.sh full` |
| `check` | 执行健康检查 | `./robot.sh check lidar` |
| `save-map` | 保存当前地图 | `./robot.sh save-map my_map` |
| `doctor` | 系统诊断 | `./robot.sh doctor` |
| `teleop` | 键盘遥控 | `./robot.sh teleop` |
| `stop` | 停止当前会话 | `./robot.sh stop` |

## 4. 建图

验证雷达方向：

```bash
./robot.sh sensor lidar
```

雷达建图（推荐命令）：

```bash
./robot.sh mapping lidar --real-base
./robot.sh mapping lidar --real-base --ekf-base
./robot.sh mapping lidar precision --real-base --ekf-base
```

`--fake-base` 用于无底盘时仅验证雷达和 SLAM：

```bash
./robot.sh mapping lidar --fake-base
```

保存地图：

```bash
./robot.sh save-map my_map
```

保存成功后更新 `~/ros2_maps/latest.yaml`，导航默认使用最近保存的地图。

## 5. 导航

标准导航启动：

```bash
./robot.sh navigation --real-base --ekf-base
```

显式指定地图：

```bash
./robot.sh navigation /home/robot/ros2_maps/my_map.yaml --real-base --ekf-base
```

启动后操作顺序：

1. 在 RViz 中执行 `2D Pose Estimate`，设置机器人当前位置
2. 确认激光点与地图墙体基本重合
3. 使用 `2D Goal Pose` 下发导航目标

默认导航配置使用独立恢复行为树，包含重规划、清空代价地图和旋转恢复动作。costmap 按真实车体外廓 `22cm × 15cm` 建模，允许低速倒车脱困。

## 6. 两阶段导航（回退模式）

仅在定位未就绪、TF 不稳定或日志提示 `send_goal failed` 时使用。非默认流程。

终端 1 — 启动定位：

```bash
./robot.sh navigation --real-base --ekf-base --localization-only
```

在 RViz 完成 `2D Pose Estimate`，确认激光与地图重合后，终端 2 — 启动 Nav2：

```bash
./robot.sh navigation --nav2-only
```

`--nav2-only` 启动前会检查 `/map_server`、`/amcl` 是否 active，并等待 `/amcl_pose`。

## 7. 运行时参数覆写

临时覆盖激光参数而不修改 YAML：

```bash
./robot.sh mapping lidar --real-base --lidar-inverted
./robot.sh mapping lidar --real-base --no-lidar-inverted
./robot.sh mapping lidar --real-base --lidar-reversion
./robot.sh mapping lidar --real-base --lidar-yaw-deg 90
```

## 8. 启动后的预期状态

建图模式下：

- `slam_toolbox` 持续发布 `/map`
- 雷达驱动发布 `/scan`
- `base_link → laser_frame` 静态 TF 存在
- 启用 `--ekf-base` 时额外发布 `/imu/data` 和 `/odometry/filtered`

导航模式下：

- AMCL 发布 `/amcl_pose` 和 `map → odom` TF
- Nav2 发布 `/plan` 和 `/local_plan`
- `stm32_bridge` 节点存在，`/odom` 持续更新
- `/cmd_vel` 有活跃发布者

## 9. 故障排查入口

- 雷达不出图：`./robot.sh check lidar`
- 底盘不动：确认 `base_mode`、`base_port` 和底盘上电状态
- 导航无响应：`./robot.sh check navigation`，确认是否已完成 `2D Pose Estimate`
- 系统诊断：`./robot.sh doctor`
- 详细排障：[07-运维与排障](./07-运维与排障.md)
