# 快速开始

这份文档只回答一件事: 如何最快把这套上位机跑起来。

当前工程的统一对外入口是 `launch_scripts/robot.sh`，技术总入口是 `src/robot_bringup/launch/system.launch.py`。如果你只想快速完成建图或导航，优先使用脚本入口；如果你要改工程结构或新增模式，优先看架构文档。

## 1. 环境要求

- Ubuntu 20.04
- ROS2 Foxy
- 已完成工作空间编译
- 传感器和底盘按当前接线接好

## 2. 构建

```bash
cd /home/robot/ros2_ws
source /opt/ros/foxy/setup.bash
colcon build --symlink-install
source /home/robot/ros2_ws/install/setup.bash
```

## 3. 最短启动路径

进入脚本目录:

```bash
cd /home/robot/ros2_ws/launch_scripts
```

雷达建图:

```bash
./robot.sh mapping lidar --real-base
```

底盘 EKF 融合雷达建图:

```bash
./robot.sh mapping lidar --real-base --ekf-base
```

精细雷达建图:

```bash
./robot.sh mapping lidar precision --real-base --ekf-base
```

摄像头建图:

```bash
./robot.sh mapping camera
```

保存地图:

```bash
./robot.sh save-map my_map
```

保存成功后会更新 `~/ros2_maps/latest.yaml`，真实底盘导航可以直接使用最近保存的地图:

```bash
./robot.sh navigation --real-base --ekf-base
```

也可以显式指定地图:

```bash
./robot.sh navigation /home/robot/ros2_maps/my_map.yaml --real-base
```

真实底盘导航并启用底盘 EKF:

```bash
./robot.sh navigation /home/robot/ros2_maps/my_map.yaml --real-base --ekf-base
```

如果 Nav2 在初始定位前启动后报 `send_goal failed`，改用两阶段导航。终端 1:

```bash
./robot.sh navigation --real-base --ekf-base --localization-only
```

RViz 里用 `2D Pose Estimate` 完成定位后，终端 2:

```bash
./robot.sh navigation --nav2-only
```

虚拟底盘导航联调:

```bash
./robot.sh navigation /home/robot/ros2_maps/my_map.yaml --fake-base
```

## 4. 你应该看到什么

- 建图时会启动 `slam_toolbox`
- 雷达模式下会发布 `/scan`
- 摄像头模式下会发布 `/camera/scan`
- 导航时会启动定位和 `Nav2`
- 真实底盘模式下会有 `/cmd_vel -> stm32_robot_bridge -> /odom` 的闭环
- 启用 `--ekf-base` 时会额外发布 `/imu/data` 和 `/odometry/filtered`
- RViz 中先用 `2D Pose Estimate` 设置当前位置，再用 `2D Goal Pose` 点目标导航

## 5. 出问题时先做什么

- 先看 [运维与排障](./07-运维与排障.md)
- 先跑 `./robot.sh doctor`
- 雷达不出图时先跑 `./robot.sh check lidar`
- 底盘不动时先确认 `base_mode` 和 `base_port`
- 精细建图不更新时先退回 `quality`，再检查 `/odom`、`/imu/data`、`/odometry/filtered`
