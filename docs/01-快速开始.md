# 快速开始

这份文档只回答一件事：如何按当前代码的正式流程把系统跑起来。

当前对外唯一推荐入口是 `launch_scripts/robot.sh`。`system.launch.py` 是内部编排入口，不作为普通使用者的主启动方式。

## 1. 环境要求

- Ubuntu 20.04
- ROS2 Foxy
- 已完成工作空间编译
- 传感器和底盘按当前接线接好

## 2. 构建

```bash
cd /home/robot/ros2_ws
source /opt/ros/foxy/setup.bash
colcon build --symlink-install
source /home/robot/ros2_ws/install/setup.bash
```

## 3. 标准启动流程

进入脚本目录：

```bash
cd /home/robot/ros2_ws/launch_scripts
```

先只验雷达方向时：

```bash
./robot.sh sensor lidar
```

雷达建图：

```bash
./robot.sh mapping lidar --real-base
./robot.sh mapping lidar --real-base --ekf-base
./robot.sh mapping lidar precision --real-base --ekf-base
```

摄像头建图：

```bash
./robot.sh mapping camera
```

保存地图：

```bash
./robot.sh save-map my_map
```

保存成功后会更新 `~/ros2_maps/latest.yaml`，导航默认可以直接使用最近保存的地图。

标准导航启动：

```bash
./robot.sh navigation --real-base --ekf-base
```

也可以显式指定地图：

```bash
./robot.sh navigation /home/robot/ros2_maps/my_map.yaml --real-base --ekf-base
./robot.sh navigation /home/robot/ros2_maps/my_map.yaml --fake-base
```

## 4. 标准导航操作

`./robot.sh navigation ...` 是当前正式主流程。运行后按下面顺序操作：

1. 先在 RViz 中执行 `2D Pose Estimate`
2. 确认激光和地图墙体基本重合
3. 再用 `2D Goal Pose` 下发目标

当前导航默认使用独立的恢复行为树，包含重规划、清空代价地图和基础恢复动作，不再复用自动建图的极简 BT。

## 5. 什么时候才用两阶段导航

只有在定位未就绪、现场 TF 不稳定、或日志明确提示 `send_goal failed` / `map->odom` 未建立时，再使用两阶段回退：

终端 1：

```bash
./robot.sh navigation --real-base --ekf-base --localization-only
```

在 RViz 完成 `2D Pose Estimate`，并确认激光与地图重合后，终端 2：

```bash
./robot.sh navigation --nav2-only
```

`--nav2-only` 启动前会先检查 `/map_server`、`/amcl` 是否 active，并等待 `/amcl_pose`，未完成定位时会直接拒绝启动。

## 6. 常用运行时覆写

如果需要临时覆盖激光参数而不改 YAML：

```bash
./robot.sh mapping lidar --real-base --lidar-inverted
./robot.sh mapping lidar --real-base --no-lidar-inverted
./robot.sh mapping lidar --real-base --lidar-reversion
./robot.sh mapping lidar --real-base --lidar-yaw-deg 90
```

## 7. 你应该看到什么

- 建图时会启动 `slam_toolbox`
- 雷达模式下会发布 `/scan`
- 导航时会同时启动定位链和 Nav2
- 真实底盘模式下会形成 `/cmd_vel -> stm32_robot_bridge -> /odom` 闭环
- 启用 `--ekf-base` 时会额外发布 `/imu/data` 和 `/odometry/filtered`
- 在 `Fixed Frame=base_link` 且只看 `LaserScan` 时，左侧物体应显示在左侧

## 8. 出问题时先做什么

- 先看 [07-运维与排障](./07-运维与排障.md)
- 先跑 `./robot.sh doctor`
- 雷达不出图时先跑 `./robot.sh check lidar`
- 底盘不动时先确认 `base_mode`、`base_port` 和底盘上电状态
- 导航异常时先确认是否已经做完 `2D Pose Estimate`
