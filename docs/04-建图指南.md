# 建图指南

当前工程支持两条建图路径：

- 雷达建图
- 摄像头建图

两条路径最终都走到 `slam_toolbox`，但正式操作入口都统一到 `./robot.sh mapping ...`。

## 1. 雷达建图

推荐命令：

```bash
cd /home/robot/ros2_ws/launch_scripts
./robot.sh mapping lidar --real-base
./robot.sh mapping lidar --real-base --ekf-base
./robot.sh mapping lidar precision --real-base --ekf-base
```

如果底盘没接：

```bash
./robot.sh mapping lidar --fake-base
```

如果只在排障时临时跳过雷达健康检查：

```bash
./robot.sh mapping lidar --skip-lidar-check
```

如果要临时覆写激光参数而不改 YAML：

```bash
./robot.sh mapping lidar --real-base --lidar-inverted
./robot.sh mapping lidar --real-base --no-lidar-inverted
./robot.sh mapping lidar --real-base --lidar-reversion
./robot.sh mapping lidar --real-base --no-lidar-reversion
./robot.sh mapping lidar --real-base --lidar-yaw-deg 90
```

当前默认雷达参数文件是 `src/robot_bringup/config/ydlidar_X2_mapping.yaml`，默认使用 `inverted: true` 修复 `/scan` 手性问题。

## 2. 摄像头建图

```bash
./robot.sh mapping camera
```

摄像头路径会经过：

- 深度相机
- `depthimage_to_laserscan`
- `/camera/scan`
- `slam_toolbox`

## 3. 建图时的底盘模式

- `real` 表示真实底盘
- `fake` 表示虚拟底盘
- `none` 表示不接底盘

`--ekf-base` 仅在 `--real-base` 下生效，启用后由 `robot_localization` 负责发布 `odom -> base_link`。

## 4. 保存地图

建图过程中保持系统运行，然后执行：

```bash
./robot.sh save-map my_map
```

地图默认保存到 `~/ros2_maps`。保存脚本会自动给地图四周添加默认 `1.0m` 的 unknown 边界缓冲，减少小地图贴边导航时的 `worldToMap failed`。

如果需要调整缓冲距离：

```bash
MAP_PADDING_METERS=1.5 ./robot.sh save-map my_map
```

如果本次修复前曾在错误激光手性下建过图，旧地图应直接废弃并重建。

## 5. 保存后导航

正式导航主流程：

```bash
./robot.sh navigation --real-base --ekf-base
```

或者显式指定地图：

```bash
./robot.sh navigation /home/robot/ros2_maps/my_map.yaml --real-base --ekf-base
```

运行后：

1. 先用 `2D Pose Estimate` 设置机器人当前位置
2. 确认激光与地图墙体基本重合
3. 再用 `2D Goal Pose` 下发目标

默认导航使用独立恢复 BT，包含重规划、清空 local/global costmap 和基础恢复动作。

## 6. 什么时候才回退到两阶段导航

只有在日志出现 `send_goal failed`、`Timed out waiting for transform from base_link to map`、或现场定位链尚未就绪时，再使用两阶段流程：

```bash
./robot.sh navigation --real-base --ekf-base --localization-only
```

完成 `2D Pose Estimate` 后：

```bash
./robot.sh navigation --nav2-only
```

两阶段是诊断和回退流程，不是当前默认主流程。

## 7. 建图与导航判断标准

- 雷达模式应看到 `/scan`
- 摄像头模式应看到 `/camera/scan`
- `slam_toolbox` 应持续发布 `/map`
- 导航时应同时存在定位链和 Nav2
- 启用 `--ekf-base` 时应看到 `/imu/data` 和 `/odometry/filtered`
- 在 `Fixed Frame=base_link` 且只看 `LaserScan` 时，左侧实物应显示在左侧

## 8. 记录建议

参数 A/B 对照时，建议录包并写记录，不要只凭体感记忆。完整记录流程见 [05-建图测试流程](./05-建图测试流程.md)。
