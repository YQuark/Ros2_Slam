# 建图指南

当前工程支持两条建图路径:

- 雷达建图
- 摄像头建图

两条路径最终都走到 `slam_toolbox`，但前置传感器和参数不同。

## 1. 雷达建图

推荐命令:

```bash
cd /home/robot/ros2_ws/launch_scripts
./robot.sh mapping lidar --real-base
```

如果需要底盘 EKF 融合:

```bash
./robot.sh mapping lidar --real-base --ekf-base
```

如果要使用精细档:

```bash
./robot.sh mapping lidar precision --real-base --ekf-base
```

如果底盘暂时没接:

```bash
./robot.sh mapping lidar --fake-base
```

如果要跳过雷达健康检查，只在排障时使用:

```bash
./robot.sh mapping lidar --skip-lidar-check
```

如果要在建图时临时覆写激光参数而不改 YAML:

```bash
./robot.sh mapping lidar --real-base --lidar-inverted
./robot.sh mapping lidar --real-base --no-lidar-inverted
./robot.sh mapping lidar --real-base --lidar-reversion
./robot.sh mapping lidar --real-base --no-lidar-reversion
./robot.sh mapping lidar --real-base --lidar-yaw-rad 1.570796326795
./robot.sh mapping lidar --real-base --lidar-yaw-deg 90
```

当前默认雷达建图参数文件是 `src/robot_bringup/config/ydlidar_X2_mapping.yaml`，默认使用 `inverted: true` 修复 `/scan` 相对 `base_link` 的左右镜像问题。按当前约定，优先只改 `inverted`，不要同时再改 `reversion` 造成双重翻转。

## 2. 摄像头建图

推荐命令:

```bash
./robot.sh mapping camera
```

摄像头模式会走:

- 深度相机
- `depthimage_to_laserscan`
- `/camera/scan`
- `slam_toolbox`

## 3. 建图时的 base 模式

- `real` 表示接入真实底盘里程计
- `fake` 表示使用虚拟底盘
- `none` 表示不接底盘，适合纯传感器建图或调试

### `--ekf-base` 的作用

- 仅在 `--real-base` 下生效
- 启用后由底盘桥接发布 `/imu/data`
- 使用 `robot_localization` 将 `/odom` 与 `/imu/data` 融合为 `/odometry/filtered`
- 启用后由 EKF 负责发布 `odom -> base_link`，桥接节点不再重复发布同名 TF

### 选择建议

- 有真实底盘就用 `real`
- 底盘未接但想联调导航就用 `fake`
- 只验证传感器和 SLAM 就用 `none`

## 4. 建图过程中的判断标准

- 雷达模式应看到 `/scan`
- 摄像头模式应看到 `/camera/scan`
- `slam_toolbox` 应该持续发布 `/map`
- RViz 中地图应逐步累积
- 启用 `--ekf-base` 时应看到 `/imu/data` 和 `/odometry/filtered`
- 在 `Fixed Frame=base_link` 且只看 `LaserScan` 时，左侧实物应显示在左侧

## 5. 保存地图

建图过程中保持系统运行，然后执行:

```bash
./robot.sh save-map my_map
```

地图默认保存到 `~/ros2_maps`。

保存脚本会自动给地图四周添加 `1.0m` 的 unknown 边界缓冲，避免小地图贴边导航时 Nav2 报 `worldToMap failed`。如果需要调整缓冲距离，可以临时设置:

```bash
MAP_PADDING_METERS=1.5 ./robot.sh save-map my_map
```

保存成功后会生成:

- `~/ros2_maps/my_map.yaml`
- `~/ros2_maps/my_map.pgm`
- `~/ros2_maps/latest.yaml`
- `~/ros2_maps/latest.pgm`

`latest.*` 会指向最近一次保存的地图，导航默认可以直接使用它。

如果本次修复前曾用错误手性的 `/scan` 建过图，旧地图应直接废弃，不能继续用于定位或导航。

## 6. 保存地图后定点导航

第一阶段先使用 RViz 作为操作面板，确认定位和 Nav2 控制链路稳定。

### 启动导航

如果刚执行过 `save-map`，可以直接使用默认地图:

```bash
./robot.sh navigation --real-base --ekf-base
```

也可以显式指定某一张地图:

```bash
./robot.sh navigation /home/robot/ros2_maps/my_map.yaml --real-base --ekf-base
```

如果日志里出现 `Timed out waiting for transform from base_link to map`、`send_goal failed`，说明 Nav2 在 AMCL 初始定位前启动，costmap 可能卡在缺少 `map->odom` 的状态。现场优先使用两阶段启动:

终端 1 先启动定位阶段:

```bash
./robot.sh navigation --real-base --ekf-base --localization-only
```

在 RViz 完成 `2D Pose Estimate`，确认雷达点和地图墙体重合后，终端 2 再启动 Nav2:

```bash
./robot.sh navigation --nav2-only
```

### 设置机器人当前位置

导航启动后，AMCL 不会自动知道机器人在保存地图里的初始位置。每次重新开机、重启导航、或手动搬动机器人后，都要在 RViz 里重新设一次初始位姿。

操作步骤:

1. 在 RViz 工具栏选择 `2D Pose Estimate`
2. 在地图上点击机器人当前实际位置
3. 按住并拖出机器人车头朝向
4. 让机器人原地小幅转动或前后移动一下
5. 确认雷达扫描和地图墙体基本重合

只有雷达扫描和地图重合后，后续点目标导航才可信。

### 点击目标点导航

定位收敛后，在 RViz 工具栏选择 `2D Goal Pose`，在地图空旷可达位置点击并拖出目标朝向。RViz 会发布目标到 `/goal_pose`，Nav2 会规划路径并通过 `/cmd_vel` 控制底盘。

如果目标太靠墙、在障碍物内、或在未知区域，Nav2 可能拒绝目标或中途恢复失败。先用 0.5-1m 的短目标验证，再测试跨房间或转弯目标。

## 7. 测试记录

建议每次参数 A/B 对照都录包并写记录，不要凭体感记忆。

完整流程见 [05-建图测试流程](./05-建图测试流程.md)。

先开一个终端启动系统，再开另一个终端执行:

```bash
cd /home/robot/ros2_ws/launch_scripts
./record_mapping_test.sh baseline --notes "./robot.sh mapping lidar --real-base"
```

脚本会自动:

- 创建时间戳测试目录
- 生成 `TEST_RECORD.md` 记录模板
- 生成 `metadata.json`
- 开始录制 `/scan /odom /imu/data /tf /tf_static /odometry/filtered`

推荐最少做两组:

```bash
./record_mapping_test.sh baseline --notes "./robot.sh mapping lidar --real-base"
./record_mapping_test.sh status-yaw --notes "BASE_USE_STATUS_YAW=true ./robot.sh mapping lidar --real-base"
```

如果要验证 EKF，再增加两组:

```bash
./record_mapping_test.sh ekf --notes "./robot.sh mapping lidar --real-base --ekf-base"
./record_mapping_test.sh ekf-status-yaw --notes "BASE_USE_STATUS_YAW=true ./robot.sh mapping lidar --real-base --ekf-base"
```

## 8. 建图建议

- 先让机器人慢速移动，再保存地图
- 雷达健康检查通过后再开始建图
- 雷达安装位姿默认按 `base_link -> laser_frame = (x=0.07, y=0.0, z=0.13, rpy=0,0,0)` 发布；如果实装位置不同，应先改传感器外参
- 雷达左右镜像优先检查 `inverted`，其次再做 `reversion` 和 `lidar_tf_yaw` 的 A/B 测试；不要先从 odom 符号链排查
- `precision` 档当前策略是保持 `0.03m` 分辨率，其余匹配门限尽量贴近 `quality` 以避免建图停滞
- 如果 RViz 显卡有问题，先看软件渲染模式和运维文档
