# 建图指南

当前工程支持两条建图路径:

- 雷达建图
- 摄像头建图

两条路径最终都走到 `slam_toolbox`，但前置传感器和参数不同。

## 1. 雷达建图

推荐命令:

```bash
cd /home/robot/ros2_ws/launch_scripts
./robot.sh mapping lidar --real-base
```

如果底盘暂时没接:

```bash
./robot.sh mapping lidar --fake-base
```

如果要跳过雷达健康检查，只在排障时使用:

```bash
./robot.sh mapping lidar --skip-lidar-check
```

## 2. 摄像头建图

推荐命令:

```bash
./robot.sh mapping camera
```

摄像头模式会走:

- 深度相机
- `depthimage_to_laserscan`
- `/camera/scan`
- `slam_toolbox`

## 3. 建图时的 base 模式

- `real` 表示接入真实底盘里程计
- `fake` 表示使用虚拟底盘
- `none` 表示不接底盘，适合纯传感器建图或调试

### 选择建议

- 有真实底盘就用 `real`
- 底盘未接但想联调导航就用 `fake`
- 只验证传感器和 SLAM 就用 `none`

## 4. 建图过程中的判断标准

- 雷达模式应看到 `/scan`
- 摄像头模式应看到 `/camera/scan`
- `slam_toolbox` 应该持续发布 `/map`
- RViz 中地图应逐步累积

## 5. 保存地图

建图过程中保持系统运行，然后执行:

```bash
./robot.sh save-map my_map
```

地图默认保存到 `~/ros2_maps`。

## 6. 建图建议

- 先让机器人慢速移动，再保存地图
- 雷达健康检查通过后再开始建图
- 如果 RViz 显卡有问题，先看软件渲染模式和运维文档
