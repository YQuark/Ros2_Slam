# 建图指南

本文档说明雷达建图的完整流程、SLAM Toolbox 参数档选择、地图保存机制和自主探索建图模式。

## 1. 雷达建图

所有建图操作统一通过 `robot.sh mapping` 入口。

### 基本命令

```bash
cd /home/robot/ros2_ws/launch_scripts

# 真实底盘建图
./robot.sh mapping lidar --real-base

# 真实底盘 + EKF 融合
./robot.sh mapping lidar --real-base --ekf-base

# 精细建图
./robot.sh mapping lidar precision --real-base --ekf-base

# 无底盘（仅验证雷达和 SLAM）
./robot.sh mapping lidar --fake-base
```

### 启动的节点

建图模式启动以下组件：

- `ydlidar_ros2_driver_node` — 发布 `/scan_raw`
- `scan_normalizer.py` — `/scan_raw → /scan`
- `static_tf_pub_laser` — `base_link → laser_frame`
- `slam_toolbox` — 发布 `/map`
- 底盘桥接（`--real-base` 时）
- EKF 融合（`--ekf-base` 时）
- RViz

### 雷达参数覆写

临时修改激光参数而不修改 YAML 文件：

```bash
./robot.sh mapping lidar --real-base --lidar-inverted
./robot.sh mapping lidar --real-base --no-lidar-inverted
./robot.sh mapping lidar --real-base --lidar-reversion
./robot.sh mapping lidar --real-base --lidar-yaw-deg 90
```

默认雷达参数文件：`src/robot_bringup/config/ydlidar_x2.yaml`，默认 `inverted: true`。

### 跳过雷达健康检查

排障时临时跳过：

```bash
./robot.sh mapping lidar --skip-lidar-check
```

## 2. SLAM Toolbox 参数档

通过 `--profile` 参数选择 SLAM 参数档：

| 档位 | 分辨率 | 地图更新间隔 | 扫描缓冲区 | 最大激光距离 | 适用场景 |
|------|--------|-------------|-----------|-------------|----------|
| `mapping`（默认） | 0.05m | 0.25s | 10 | 6.0m | 标准建图 |
| `fast` | 0.08m | 0.10s | 6 | 8.0m | 快速建图，降低精度 |
| `realtime` | 0.10m | 0.05s | 4 | 3.5m | 实时模式 |
| `precision` | 0.03m | 0.25s | 10 | 6.0m | 精细建图 |

参数档文件位于 `src/robot_bringup/config/slam_toolbox_mapping*.yaml`。

`robot.sh` 的 `choose_slam_params()` 函数根据 `--profile` 参数选择对应的 YAML 文件。

## 3. 底盘模式

| 模式 | 参数 | 说明 |
|------|------|------|
| 真实底盘 | `--real-base` | 启动 `stm32_bridge`，发布 `/odom` 和 `odom → base_link` |
| 虚拟底盘 | `--fake-base` | 启动 `fake_base_odom.py`，发布静态 TF |
| 无底盘 | 不指定 | 无里程计，需 `fallback_static_odom:=true` |

`--ekf-base` 仅在 `--real-base` 下生效。启用后：

- `stm32_bridge` 的 `publish_tf` 自动设为 `false`
- `stm32_bridge` 的 `publish_imu` 自动设为 `true`
- `ekf_node` 发布 `/odometry/filtered` 和 `odom → base_link`

## 4. 地图保存

建图过程中保持系统运行，另开终端执行：

```bash
./robot.sh save-map my_map
```

保存机制：

- 地图默认保存到 `~/ros2_maps/`
- 脚本自动给地图四周添加 `1.0m` 的 unknown 边界缓冲，减少小地图贴边导航时的 `worldToMap failed`
- 保存后更新 `~/ros2_maps/latest.yaml`

调整缓冲距离：

```bash
MAP_PADDING_METERS=1.5 ./robot.sh save-map my_map
```

## 5. 自主探索建图

通过 `auto_mapping_drive:=true` 启用自主探索模式，系统额外启动：

- `frontier_explorer.py` — 边界探索节点
- Nav2 导航栈 — 用于路径规划和导航

```bash
./robot.sh mapping lidar --real-base --auto-mapping-drive
```

### frontier_explorer.py 工作流程

1. 启动后等待 `startup_delay_sec`（默认 5s），让 SLAM 初始建图稳定
2. 每隔 `tick_hz`（默认 2Hz）检查 `/map` 更新
3. 在地图中搜索边界（frontier）—— 已知区域与未知区域的交界
4. 过滤条件：
   - 边界簇最小格数 ≥ `min_frontier_cluster_cells`（默认 10）
   - 距离机器人在 `frontier_min_distance`（0.70m）到 `frontier_max_distance`（2.5m）之间
5. 选择最近的有效边界作为导航目标
6. 目标点向已知区域偏移 `goal_approach_offset`（0.55m），避免导航到未知区域
7. 通过 NavigateToPose action 发送目标
8. 安全约束：
   - `goal_clearance_radius`（0.38m）— 目标点周围障碍物清空半径
   - `goal_unknown_clearance_radius`（0.35m）— 目标点周围未知区域清空半径
   - `blacklist_radius`（0.35m）— 导航失败后拉黑周围区域，TTL 30s
9. 达到 `max_duration_sec`（180s）或无可用边界时结束

### 行为树

自主探索使用 `mapping_navigate_to_pose.xml`，结构：

```text
RecoveryNode(retries=3)
  PipelineSequence
    RateController(1Hz)
      RecoveryNode(retries=1)
        ComputePathToPose(planner_id=GridBased)
        ClearEntireCostmap(global)
    RecoveryNode(retries=1)
      FollowPath(controller_id=FollowPath)
      ClearEntireCostmap(local)
  Spin(1.57rad)
```

## 6. 建图与导航的判断标准

建图模式下应验证：

- `/scan` 持续发布且频率稳定
- `slam_toolbox` 持续发布 `/map`
- 启用 `--ekf-base` 时 `/imu/data` 和 `/odometry/filtered` 持续更新
- 在 `Fixed Frame=base_link` 且只看 `LaserScan` 时，左侧实物应显示在左侧

## 7. 建图后导航

```bash
./robot.sh navigation --real-base --ekf-base
```

或显式指定地图：

```bash
./robot.sh navigation /home/robot/ros2_maps/my_map.yaml --real-base --ekf-base
```

启动后在 RViz 中依次执行 `2D Pose Estimate` 和 `2D Goal Pose`。
