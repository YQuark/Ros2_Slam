# 建图指南

当前工程支持两条建图路径:

- 雷达建图
- 摄像头建图

两条路径最终都走到 `slam_toolbox`，但前置传感器和参数不同。

## 1. 雷达建图

推荐命令:

```bash
cd /home/robot/ros2_ws/launch_scripts
./robot.sh mapping lidar --real-base
```

如果需要底盘 EKF 融合:

```bash
./robot.sh mapping lidar --real-base --ekf-base
```

如果要使用精细档:

```bash
./robot.sh mapping lidar precision --real-base --ekf-base
```

如果要让 `/odom.pose.yaw` 直接使用下位机当前融合航向:

```bash
./robot.sh mapping lidar --real-base --base-use-status-yaw
```

如果底盘暂时没接:

```bash
./robot.sh mapping lidar --fake-base
```

如果要跳过雷达健康检查，只在排障时使用:

```bash
./robot.sh mapping lidar --skip-lidar-check
```

## 2. 摄像头建图

推荐命令:

```bash
./robot.sh mapping camera
```

摄像头模式会走:

- 深度相机
- `depthimage_to_laserscan`
- `/camera/scan`
- `slam_toolbox`

## 3. 建图时的 base 模式

- `real` 表示接入真实底盘里程计
- `fake` 表示使用虚拟底盘
- `none` 表示不接底盘，适合纯传感器建图或调试

### `--ekf-base` 的作用

- 仅在 `--real-base` 下生效
- 启用后由底盘桥接发布 `/imu/data`
- 使用 `robot_localization` 将 `/odom` 与 `/imu/data` 融合为 `/odometry/filtered`
- 启用后由 EKF 负责发布 `odom -> base_link`，桥接节点不再重复发布同名 TF

### 选择建议

- 有真实底盘就用 `real`
- 底盘未接但想联调导航就用 `fake`
- 只验证传感器和 SLAM 就用 `none`

## 4. 建图过程中的判断标准

- 雷达模式应看到 `/scan`
- 摄像头模式应看到 `/camera/scan`
- `slam_toolbox` 应该持续发布 `/map`
- RViz 中地图应逐步累积
- 启用 `--ekf-base` 时应看到 `/imu/data` 和 `/odometry/filtered`

## 5. 保存地图

建图过程中保持系统运行，然后执行:

```bash
./robot.sh save-map my_map
```

地图默认保存到 `~/ros2_maps`。

## 6. 测试记录

建议每次参数 A/B 对照都录包并写记录，不要凭体感记忆。

完整流程见 [05-建图测试流程](./05-建图测试流程.md)。

先开一个终端启动系统，再开另一个终端执行:

```bash
cd /home/robot/ros2_ws/launch_scripts
./record_mapping_test.sh baseline --notes "./robot.sh mapping lidar --real-base"
```

脚本会自动:

- 创建时间戳测试目录
- 生成 `TEST_RECORD.md` 记录模板
- 生成 `metadata.json`
- 开始录制 `/scan /odom /imu/data /tf /tf_static /odometry/filtered`

推荐最少做两组:

```bash
./record_mapping_test.sh baseline --notes "./robot.sh mapping lidar --real-base"
./record_mapping_test.sh status-yaw --notes "./robot.sh mapping lidar --real-base --base-use-status-yaw"
```

如果要验证 EKF，再增加两组:

```bash
./record_mapping_test.sh ekf --notes "./robot.sh mapping lidar --real-base --ekf-base"
./record_mapping_test.sh ekf-status-yaw --notes "./robot.sh mapping lidar --real-base --ekf-base --base-use-status-yaw"
```

## 7. 建图建议

- 先让机器人慢速移动，再保存地图
- 雷达健康检查通过后再开始建图
- 雷达安装位姿默认按 `base_link -> laser_frame = (x=0.07, y=0.0, z=0.13, rpy=0,0,0)` 发布；如果实装位置不同，应先改传感器外参
- `precision` 档当前策略是保持 `0.03m` 分辨率，其余匹配门限尽量贴近 `quality` 以避免建图停滞
- 如果 RViz 显卡有问题，先看软件渲染模式和运维文档
