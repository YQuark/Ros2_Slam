# 串口桥接协议

本文档描述 `stm32_robot_bridge` 包实现的 v1 串口协议，包括帧格式、命令定义、ROS2 话题映射、错误处理机制和节点参数。

协议定义以下位机代码为权威来源：
- 下位机：`App/protocol/upper_protocol.h`（C 结构体和常量）
- 下位机文档：`docs/protocols/2026-04-27-usart3-upper-protocol.md`

---

## 1. 物理层

| 参数 | 值 |
|------|-----|
| 接口 | USB-UART (CP2102) |
| 波特率 | 115200 |
| 数据位 | 8 |
| 校验 | None |
| 停止位 | 1 |
| 设备名 | `/dev/stm32_chassis`（udev 规则）或自动探测 |

## 2. 帧格式

所有帧共享统一格式：

```text
┌──────────┬──────────┬────────┬─────┬─────────┬──────────┐
│ SOF (1B) │ SOF2(1B) │ Len(1B)│Cmd  │ Payload │  CS (1B) │
│   0xA5   │   0x5A   │        │(1B) │ (Len B) │          │
└──────────┴──────────┴────────┴─────┴─────────┴──────────┘
```

- **SOF**：帧头标志，固定 `0xA5 0x5A`
- **Length**：Payload 长度（不含帧头、Length、Cmd、CS）
- **Cmd**：命令字
- **Checksum8**：`(Length + Cmd + sum(Payload)) & 0xFF`

## 3. 命令定义

### 3.1 SET_VELOCITY (0x01) — 控制帧

上位机发送，控制底盘运动。

```text
Payload (10 字节):
  float  linear_x   — 线速度 (m/s)，正值前进
  float  angular_z   — 角速度 (rad/s)，正值逆时针
  uint8  enable      — 1=使能, 0=禁用
  uint8  mode        — 0=idle, 1=open_loop, 2=closed_loop
```

上位机默认发送 `enable=1, mode=2(closed_loop)`。`/cmd_vel` 超时后发送零速。

### 3.2 ESTOP (0x02) — 急停帧

```text
Payload (1 字节):
  uint8  enabled     — 1=触发急停, 0=解除急停
```

当前 bridge_node 未实现 ESTOP 订阅，预留命令字。

### 3.3 STATUS (0x81) — 状态帧

下位机响应上位机的 STATUS 请求，或主动上报。

```text
Payload (47 字节):
  float   left_speed        — 左轮速 (m/s)
  float   right_speed       — 右轮速 (m/s)
  int32   left_encoder      — 左编码器累积计数
  int32   right_encoder     — 右编码器累积计数
  float   battery_voltage   — 电池电压 (V)
  float   left_current      — 左电机电流 (A)
  float   right_current     — 右电机电流 (A)
  int16   imu_accel[3]      — 加速度计原始值 (x, y, z)
  int16   imu_gyro[3]       — 陀螺仪原始值 (x, y, z)
  uint32  error_flags       — 错误标志位
  uint8   control_mode      — 当前控制模式
```

**IMU 换算公式**：

- 加速度：`real_m_s2 = raw * 16.0 * 9.80665 / 32768`（±16g 量程）
- 角速度：`real_rad_s = raw * 2000.0 * π / (180 * 32768)`（±2000°/s 量程）

**control_mode 枚举**：

| 值 | 含义 |
|----|------|
| 0 | IDLE — 空闲 |
| 1 | OPEN_LOOP — 开环 |
| 2 | CLOSED_LOOP — 闭环（正常运行要求此模式） |

**error_flags 位定义**：各 bit 含义由下位机定义，上位机以原始 uint32 发布到 `/chassis/status`，不做解析。

## 4. 通信流程

```text
上位机                          下位机
  │                               │
  ├── SET_VELOCITY (周期 50ms) ──→│
  ├── STATUS 请求 (周期 50ms) ───→│
  │                               │
  │←──── STATUS 响应 ─────────────┤
  │                               │
```

- **控制频率**：`control_hz=20Hz`，每 50ms 发送一次 SET_VELOCITY
- **状态轮询**：`status_hz=20Hz`，每 50ms 发送一次 STATUS 请求
- **保活机制**：`drive_keepalive_sec=0.10s`，即使速度不变也至少每 100ms 重发

## 5. ROS2 话题

### 发布

| 话题 | 消息类型 | 说明 | 发布条件 |
|------|----------|------|----------|
| `/odom` | `nav_msgs/Odometry` | 里程计（轮速计算） | 每次收到 STATUS |
| `/tf` | `geometry_msgs/TransformStamped` | odom→base_link | publish_tf=true 时 |
| `/imu/data` | `sensor_msgs/Imu` | IMU 数据 | publish_imu=true 时 |
| `/chassis/status` | `std_msgs/UInt32` | error_flags 原始值 | 每次收到 STATUS |
| `/battery_state` | `sensor_msgs/BatteryState` | 电池电压 | 每次收到 STATUS |
| `/motor/left_current` | `std_msgs/Float32` | 左电机电流 | 每次收到 STATUS |
| `/motor/right_current` | `std_msgs/Float32` | 右电机电流 | 每次收到 STATUS |

### 订阅

| 话题 | 消息类型 | 说明 |
|------|----------|------|
| `/cmd_vel` | `geometry_msgs/Twist` | 速度指令 |

## 6. 里程计计算

bridge_node 从 STATUS 帧的 `left_speed` / `right_speed`（m/s）直接计算里程计：

```text
vx = (left_speed + right_speed) / 2
wz = (right_speed - left_speed) / wheel_track_width
```

- `wheel_track_width` 默认 0.1250m
- `odom_linear_scale` / `odom_angular_scale`：缩放校正系数
- `odom_angular_sign`：角速度符号翻转（用于轮子安装方向修正）
- `odom_linear_deadzone` / `odom_angular_deadzone`：低于阈值置零

**状态超时处理**：当 STATUS 帧超过 `status_timeout=0.75s` 未更新时：
- 里程计速度置零（保持位姿不变）
- 协方差膨胀至 `[1.0, 1.0, 3.14, 1000.0, 1000.0]`

## 7. 串口自动探测

当 `port=auto` 时，bridge_node 执行以下探测流程：

1. `detect_cp210x_ports()` 扫描 `/dev/serial/by-id/` 和 `/dev/ttyUSB*`，识别 CP210x 芯片
2. 排除 `excluded_ports` 中的设备（避免与雷达串口冲突）
3. 对每个候选端口发送 STATUS 请求（`probe_port_for_status`），验证收到 47 字节 STATUS 响应
4. 使用第一个通过探测的端口

探测失败时 bridge_node 以 `serial_open_retry_sec=2.0s` 间隔持续重试。

## 9. 错误处理

| 错误场景 | 处理方式 |
|----------|----------|
| 串口打开失败 | 每隔 `serial_open_retry_sec` 重试，日志告警 |
| 串口读写异常 | 关闭串口，触发重新探测 |
| STATUS 帧超时 (>0.75s) | 里程计速度置零，协方差膨胀 |
| 帧校验失败 | 丢弃该帧，继续搜索下一个帧头 |
| `/cmd_vel` 超时 (>0.25s) | 发送零速 SET_VELOCITY |
| 非 closed_loop 模式 | 日志告警（每 2 秒一次） |

```text

```

### bridge_node.py 核心类

**Protocol**（静态工具类）：
- `build_frame(cmd, payload)` → 构造完整帧
- `find_frame(buf)` → 在缓冲区中搜索有效帧，返回 `(cmd, payload), consumed`

**STM32Bridge(Node)**（ROS2 节点）：
- `__init__` — 参数声明、话题创建、定时器注册
- `resolve_port()` — 串口自动探测入口
- `probe_port_for_status(port)` — 向候选端口发送 STATUS 验证连通性
- `ensure_serial()` — 确保串口已打开，失败时持续重试
- `control_loop()` — 20Hz 控制循环：读串口、计算速度、发送 SET_VELOCITY、发布 odom
- `status_poll()` — 20Hz 状态轮询：读串口、发送 STATUS 请求
- `handle_frame(cmd, payload)` — 解析 STATUS 帧，更新内部状态，发布辅助话题
- `publish_odom(now)` — 从轮速计算里程计，发布 Odometry 和 TF
- `publish_imu_msg(now)` — 从 IMU 原始数据发布 Imu 消息
